# 细化操作指南

该 skill 用于把科服成员不想手动展开的操作，转化成普通电脑用户能够理解和执行的详细指引。用户通常会使用 `/explain 要解释的操作` 显式调用，也可以在明显需要细化步骤时主动采用。

## 处理原则

1. 先判断目标操作是否明确。若系统版本、设备状态、操作对象或目标不清楚，先询问一至两个会影响步骤或安全性的关键信息，不要凭空假设。
2. 用普通用户熟悉的界面名称描述路径，并给出必要的按钮、菜单、搜索词或命令。存在多个入口时优先介绍最简单、最通用的一种。
3. 将操作拆成短步骤并编号，每一步只包含一个主要动作。重要界面应说明用户预期看到什么，以及如何确认该步成功。
4. 保持即时通信风格。较长流程应先提供准备工作和第一阶段，让用户操作后反馈结果，再继续后续阶段；除非用户明确要求一次给出完整流程。
5. 如果 Windows、软件版本或设备型号会造成界面差异，要明确指出差异，并说明如何确认用户使用的版本。

## 安全要求

1. 在步骤开始前识别数据丢失、断电、误格式化、误删分区、硬件损坏、触电、烫伤和静电等风险。
2. 存在数据风险时先要求备份重要文件；存在硬件风险时先要求关机、断开电源，并在适用时做好防静电措施。
3. 明确写出停止条件，例如出现异响、异味、冒烟、异常高温、火花、鼓包、重要数据无法读取或界面与描述明显不符时立即停止。
4. 不通过“继续尝试”绕过错误或安全警告，不指导普通用户执行无法可靠回退的高风险操作。
5. 操作超出普通用户安全能力，或风险无法通过预防措施合理降低时，不继续展开危险步骤，改为指引用户在每日 19:00–22:00 到 C 楼科技服务队线下处理。

## 推荐输出结构

- 先用一句话说明该操作的作用，以及它是否可能影响数据。
- 列出开始前需要确认或准备的事项。
- 给出当前阶段的编号步骤。
- 说明完成后应看到的结果，以及需要回复给科服成员的信息。
- 必要时补充失败后的安全退出方法和停止条件。

不要只给出概念说明，也不要默认声称操作已经成功。
